# Challenge Report: GDPR Telemetry and DP Noise Adversarial Verification

## Challenge Summary

**Overall risk assessment**: HIGH

Through systematic stress-testing and boundary analysis of the C++ `edge-daemon`'s GDPR compliance metrics (specifically keystroke count and mouse pixel distance) and Laplace noise generation, multiple severe bugs and design assumptions were exposed. Crucially, the system fails to prevent the database storage of infinite values, generating invalid JSON database backups that will break downstream parsers. Additionally, boundary conditions in Laplace noise generation can produce infinite noise, and SQLite rejects NaN values due to NULL conversions, causing silent data drops.

---

## Challenges

### [High] Challenge 1: Invalid JSON Generation in Database Backups
- **Assumption challenged**: The daemon assumes that C++ stream serialization of telemetry values (represented as doubles) always produces valid JSON-compliant numbers.
- **Attack scenario**: If a telemetry event gets a value of `inf` or `-inf` (due to extremely large input parsing or Laplace noise edge cases), C++ `std::ostream` serializes this as raw `inf` or `-inf` directly into the JSON string generated by `dumpBackupToJson`.
- **Blast radius**: The backup JSON file contains invalid number formatting (e.g. `"value": inf`), violating RFC 8259 specifications. Standard JSON parsers (e.g., Python's `json.loads` or Node's `JSON.parse`) will raise syntax errors and fail to parse the entire backup file, corrupting backup recovery.
- **Mitigation**: Explicitly check floating-point sanity (`std::isfinite(value)`) before JSON serialization in `dumpBackupToJson`. Format non-finite values as strings, omit the fields, or discard the records.

### [Medium] Challenge 2: SQLITE_CONSTRAINT_NOTNULL Violation on NaN Values
- **Assumption challenged**: The daemon assumes that any anonymized telemetry value can be safely buffered via SQLite's `bufferEvent` without query failures.
- **Attack scenario**: When a telemetry event contains a `NaN` value (resulting from invalid configuration inputs or math overflows), `sqlite3_bind_double` converts the C++ `double` NaN to `NULL` internally. Since the database schema defines `value REAL NOT NULL`, the insert command fails with error code `19` (`SQLITE_CONSTRAINT_NOTNULL`).
- **Blast radius**: `bufferEvent` returns `false`, causing the telemetry event to be silently dropped and lost.
- **Mitigation**: Add checks for `std::isnan(value)` before writing to SQLite. Handle NaN records gracefully at the application layer.

### [Medium] Challenge 3: Infinite Laplace Noise Generation on Boundary Condition
- **Assumption challenged**: The inverse cumulative distribution function (CDF) for Laplace noise generation is assumed to always return finite values.
- **Attack scenario**: The uniform real distribution generates values in the range `[-0.5, 0.5)`. The lower bound `-0.5` is inclusive.
  If $u = -0.5$, the inverse CDF evaluation:
  `noise = -b * (u > 0 ? 1 : -1) * std::log(1.0 - 2.0 * std::abs(u))`
  leads to `std::log(1.0 - 2.0 * 0.5) = std::log(0.0) = -inf`.
  The function then returns `-b * (-1) * (-inf) = -inf`.
  Likewise, if float rounding causes $2.0 \times \text{abs}(u) \approx 1.0$ for $u$ close to $0.5$, it yields `+inf`.
- **Blast radius**: Non-zero probability of generating infinite noise, making the entire telemetry value infinite and triggering database insertion errors or corrupted JSON exports.
- **Mitigation**: Clamp the argument of `std::log` (e.g. `std::max(1e-15, 1.0 - 2.0 * std::abs(u))`) to guarantee finite results.

### [Low/Medium] Challenge 4: Absence of Input Validation on Epsilon and Sensitivity
- **Assumption challenged**: Differential privacy configuration parameters `epsilon` and `sensitivity` are assumed to be strictly positive finite numbers.
- **Attack scenario**: The `/control` configuration API and CLI arguments lack any validation. A user or agent can configure `epsilon = 0.0`, `epsilon = -1.0`, or `epsilon = nan`.
- **Blast radius**:
  - `epsilon = 0.0` causes division by zero, yielding infinite or NaN noise.
  - `epsilon = nan` leads to NaN noise and silent SQLite insert drops.
  - `epsilon = -1.0` is accepted, mathematically invalidating DP guarantees.
- **Mitigation**: Validate that configuration updates satisfy `epsilon > 0.0`, `sensitivity > 0.0`, and `std::isfinite`.

### [Low] Challenge 5: Lack of Output Clamping for GDPR Metrics
- **Assumption challenged**: Telemetry counts (keystrokes and mouse pixel distance) are inherently non-negative.
- **Attack scenario**: If a user performs `0` keystrokes/pixels, and the added Laplace noise is negative, the anonymized value becomes negative (e.g. `-3.28`).
- **Blast radius**: The transmitted telemetry contains negative counts, which are physically impossible and can cause errors in backend metrics aggregation.
- **Mitigation**: Implement post-anonymization lower-bound clamping (e.g., `std::max(0.0, value)`), balanced with mathematical privacy bias constraints.

## Stress Test Results

- **Laplace noise math with $u = -0.5$** → Expected: Finite noise → Actual: `-inf` → **FAIL**
- **Laplace noise math with $u \approx 0.5$** → Expected: Finite noise → Actual: `inf` → **FAIL**
- **Epsilon config = 0.0** → Expected: Config validation error → Actual: Accepted, noise generated is `inf` → **FAIL**
- **Keystrokes = 0.0 anonymization** → Expected: Non-negative value → Actual: Negative value (e.g. `-3.28`) → **FAIL**
- **SQLite NaN insert** → Expected: Successful storage → Actual: Fails with `SQLITE_CONSTRAINT_NOTNULL` (rc = 19) → **FAIL**
- **SQLite Inf insert** → Expected: Successful storage → Actual: SUCCESS → **PASS**
- **SQLite backup dump with inf/-inf** → Expected: Valid JSON file → Actual: Contains raw `inf`/`-inf` (`"value": inf`), invalid JSON → **FAIL**

## Unchallenged Areas

- **Backend Ingestion Server Validation** — The backend JSON parser and ingest logic were not tested directly due to local testing restrictions and network isolation.
- **SQLite Database Corruption** — Did not investigate potential corruption of database files under persistent disk full scenarios.
